# 安全策略

## 支持范围

安全修复优先面向仓库当前最新版本。Codex 桌面端或 app-server 协议更新造成的兼容问题不等同于安全漏洞，但欢迎通过 Issue 提交不含敏感信息的复现步骤。

## 报告安全问题

请不要在公开 Issue 中披露尚未修复的漏洞、认证信息或可被直接利用的细节。建议通过 GitHub 仓库的 Security Advisory 私密报告功能联系维护者。

报告时请尽量包含：

- 受影响版本与 macOS、Codex 版本；
- 问题影响和最小复现步骤；
- 可能的修复建议；
- 已移除 Cookie、Access Token、账户响应和其他个人数据的日志片段。

## 项目安全边界

- CDP 仅允许连接 `127.0.0.1`、`localhost` 或 `[::1]`。
- 项目不会修改 Codex 安装包或应用签名。
- 项目不会保存 Cookie、Access Token 或完整账户额度响应。
- 请勿发布已改为局域网监听、跳过目标地址校验或写入真实凭证的变体。
