# 贡献指南

感谢你愿意为 Codex Usage Injector 提交改进。

## 开始之前

- 当前项目面向 macOS 官方 Codex 桌面应用。
- 运行环境需要 Node.js 22 或更高版本。
- 注入端口必须绑定本机回环地址，禁止改为局域网或公网地址。
- 不要提交 Cookie、Access Token、账户响应、运行日志或其他敏感信息。

## 开发流程

1. Fork 仓库并从最新分支创建功能分支。
2. 保持改动聚焦，避免将无关重构混入同一次提交。
3. 修改 Renderer 定位逻辑时，优先使用稳定的结构特征，不依赖用户名或界面语言。
4. 对新增行为补充对应测试和必要的中文维护注释。
5. 提交前运行：

```bash
npm test
npm run check
```

如果修改了真实注入链路，还应在开启本机 CDP 的 Codex 中运行状态检查，确认返回 `mounted: true`。

## 提交 Pull Request

请在 PR 描述中说明：

- 改动内容和原因；
- 对用户行为的影响；
- 已执行的验证；
- 如果是兼容性修复，请附上 Codex 版本和相关 DOM 特征，但不要包含个人账户数据。
